**Lib.Models.TinyNN** — це компактна, але повноцінна нейронна мовна модель для проєкту Mini-ChatGPT. Вона здатна не лише генерувати текст, спираючись на контекст, але й **навчатися** на нових даних за допомогою градієнтного спуску (на відміну від статичних статистичних моделей).

---

### 1. Перетворення слів у вектори (**Embeddings**)

Щоб модель могла аналізувати текст, кожне слово зі словника отримує своє математичне представлення — вектор із 32 чисел (за замовчуванням).

- Коли надходить фраза (контекст), модель бере вектори всіх останніх слів (до 8 токенів) і **усереднює їх**. Це створює єдиний "смисловий вектор", який описує поточний стан розмови.

### 2. Проєкція та прогноз (**Linear Head**)

Отриманий усереднений вектор контексту пропускається через повнозв'язний лінійний шар (матричне множення + зміщення).

- Результатом є масив **логітів** розміром зі словник (`VocabSize`). Найбільше значення в цьому масиві вказує на слово, яке мережа вважає найімовірнішим продовженням.

### 3. Процес навчання (**Train Step**)

На відміну від Трансформера (на Етапі 1), ця модель має вбудований цикл навчання. Вона робить прогноз, а потім використовує математичне ядро (`Lib.MathCore`) для розрахунку ймовірностей (Softmax) та оцінки помилки (Cross-Entropy Loss), порівнюючи свій вибір із реально правильним наступним словом.

### 4. Оновлення знань (**Backward Pass**)

Дізнавшись свою помилку, модель запускає процес зворотного поширення (Backpropagation). Вона обчислює градієнти (напрямок для покращення) і трохи коригує свої "синапси" (ваги в `LinearHead` та `Embeddings`), щоб наступного разу зробити точніший прогноз.

## Опис класів в проекті

### 1. **`Configuration/TinyNNConfig.cs`**

Клас-конфігурація (record), що визначає форму нейромережі:

- **`VocabSize`**: Кількість можливих слів.
- **`EmbeddingSize`**: Довжина вектора для кожного слова (за замовчуванням **32**).
- **`ContextSize`**: Максимальна кількість слів у пам'яті (за замовчуванням **8**). Якщо слів більше, старі відкидаються.

---

### 2. **`State/TinyNNWeights.cs`**

Відповідає за збереження "пам'яті" нейромережі — її ваг.

- Містить матриці: `Embeddings`, `OutputWeights` та вектор `OutputBias`.
- **Ініціалізація**: Усі початкові ваги заповнюються випадковими числами в діапазоні **[-0.1, 0.1]**, щоб уникнути математичного переповнення на початку навчання. Адаптовано для збереження у форматі JSON.

---

### 3. **`Layers/EmbeddingLayer.cs`**

Шар перетворення тексту у вектори.

- **`EncodeContext`**: Знаходить вектори потрібних слів і вираховує їхнє середнє арифметичне.
- **`Backward`**: Отримує градієнти помилки з наступного шару і оновлює таблицю `Embeddings` з урахуванням швидкості навчання (`lr`).

---

### 4. **`Layers/LinearHead.cs`**

Фінальний шар нейромережі.

- **`Project`**: Перетворює "прихований стан" (усереднений вектор) у логіти (оцінки для кожного слова) за допомогою множення на `OutputWeights` та додавання `OutputBias`.
- **`Backward`**: Розраховує градієнти для матриці ваг та вектора зміщень, оновлює їх, а також обчислює та повертає помилку `dHidden` для передачі попередньому шару.

---

### 5. **`TinyNNModel.cs`**

Головний фасад, що реалізує інтерфейс `ILanguageModel`.

- **`NextTokenScores`**: Виконує прямий прохід (Forward pass) через `EmbeddingLayer` та `LinearHead` для генерації тексту.
- **`TrainStep`**: Повний цикл навчання (Forward -> Розрахунок Loss через `MathOps` -> Backward для обох шарів).
- **`ToPayload`**: Упаковує конфігурацію та поточний стан ваг у `TinyNNPayload` для збереження прогресу у файл (Checkpoint).

---

### 6. **`Factories/TinyNNModelFactory.cs`**

Фабрика для керування життєвим циклом моделі.

- **`CreateNew`**: Створює нову порожню модель зі згенерованими випадковими вагами.
- **`FromPayload`**: Парсить JSON-чекпоінт, десеріалізує конфігурацію та ваги, і збирає точну копію навченої моделі (Round-Trip).

---

## 🎯 Гарантії та Критерії готовності (DoD)

Модель повністю відповідає технічним стандартам системи:

1. **Інтеграція з MathCore:** Модель успішно використовує зовнішні `Softmax` та `CrossEntropyLoss` для навчання.
2. **TrainStep:** Зворотне поширення помилки реалізовано коректно; після викликів `TrainStep` показник `Loss` зменшується.
3. **Обрізання контексту:** Модель стабільно працює, безпечно відкидаючи зайві слова, якщо поданий контекст перевищує ліміт `ContextSize`.
4. **Checkpoint Round-Trip:** Повний стан мережі (`Weights` + `Config`) успішно серіалізується в JSON і відновлюється фабрикою без втрати знань.
